## 🚨 问题描述

### 修复前的问题

**skill.py 中存在硬编码公司路径：**
```python

WIKI_DIR = Path('/home/admin/.openclaw/workspace/knowledge-base/wiki')

```

**风险：**
- 🔴 技能代码中硬编码了公司知识路径
- 🔴 分享技能时会暴露公司知识结构
- 🔴 违背"系统代码与公司知识分离"原则
- 🔴 不符合 Karpathy 方法

---

## ✅ 修复方案

### 架构设计

```
修复前：
skill.py → 硬编码公司路径 → 公司知识 ❌

修复后：
skill.py → 读取 config.json → 获取路径 → 公司知识 ✅
```

### 核心改动

#### 1. skill.py - 通用代码 ✅

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
知识整理技能 - LLM 驱动的知识查询和组织

安全架构：
- 代码通用化，不包含公司特定路径
- 路径通过 config.json 配置注入
- 支持默认值，便于代码分享
"""

# 从 config.json 加载路径
SKILL_DIR = Path(__file__).parent
CONFIG_FILE = SKILL_DIR / 'config.json'

def load_config():
    if CONFIG_FILE.exists():
        config = json.loads(CONFIG_FILE.read_text(encoding='utf-8'))
        return config.get('config', {}), config.get('defaults', {})
    return {}, {}

config, defaults = load_config()

# 使用配置路径（优先）或默认路径
WIKI_DIR = Path(config.get('knowledge_base_path', 
              defaults.get('knowledge_base_path', './knowledge-base'))) / 'wiki'
COMPANY_DIR = Path(config.get('company_data_path', 
                 defaults.get('company_data_path', './wiki/company')))
```

**特点：**
- ✅ 无硬编码公司路径
- ✅ 从配置文件读取
- ✅ 有默认值便于分享
- ✅ 代码本身无敏感信息
